
MariaDB 란?

MariaDB는 MySQL의 원래 개발자들이 만든 무료 오픈소스 관계형 데이터베이스 관리 시스템(RDBMS)으로, 
MySQL과 높은 호환성을 유지하며 속도, 안정성, 확장성에 중점을 둔 데이터베이스입니다. 
MySQL의 소스 코드를 기반으로 개발되었으나, 오라클의 MySQL 인수에 따른 라이선스 불확실성을 해결하고자 만들어졌으며, 
엔터프라이즈 환경에서 요구되는 고가용성, 확장성, 그리고 다양한 워크로드를 지원하는 데 강점을 가지고 있습니다. 

[주요 특징]
오픈소스 및 MySQL 호환성:
    MySQL의 원래 개발자들이 만들었으며, MySQL과 소스 코드를 공유하므로 높은 호환성을 제공합니다. 
성능 및 확장성:
    MySQL보다 뛰어난 확장성과 빠른 쿼리 속도를 제공하며, 대용량 데이터 관리 및 고부하 처리에 적합합니다. 
다양한 기능 지원:
    시퀀스 스토리지 엔진, 가상 열, 여러 스토리지 엔진을 동시에 사용할 수 있는 기능 등 MySQL에는 없는 독자적인 기능을 제공합니다. 
높은 가용성과 안정성:
    고가용성 및 안정적인 운영을 지원하는 클라우드 네이티브 데이터베이스로 설계되었습니다. 
폭넓은 지원:
    Windows, Linux, macOS 등 모든 주요 운영 체제에서 사용 가능하며, 다양한 프로그래밍 언어(C, C#, Java, Python 등)를 지원합니다. 

[활용 분야]
트랜잭션 애플리케이션:
    빈번한 쿼리 처리와 빠른 응답 속도가 요구되는 엔터프라이즈 트랜잭션 애플리케이션에 적합합니다. 
웹 및 이커머스 플랫폼:
    웹 애플리케이션과 이커머스 플랫폼에서 원활하게 작동하며, 멀티스레딩을 통해 높은 로드를 처리할 수 있습니다. 
대규모 데이터 관리:
    대용량 데이터를 효율적으로 관리하고 처리해야 하는 환경에 유용합니다


# MySQL은 여전히 GPL 라이선스의 커뮤니티 에디션이 존재하지만, 기업용으로 사용되는 Standard, Enterprise 등은 유료 상용 라이선스가 적용됩니다. 


MySQL 란?
MySQL은 다양한 기능과 유연한 확장성을 제공하며, 초보자도 쉽게 DB 생성을 할 수 있는 사용자 친화적인 인터페이스를 갖추고 있습니다. 
MySQL Workbench 같은 시각적 도구를 사용하면 복잡한 명령어 없이도 데이터베이스를 설계하고 관리할 수 있습니다. 



MySQL vs MariaDB 성능 차이?
MySQL - 오라클의 상용 지원을 통해 안정성과 보안을 보장
        보안 패치와 업데이트가 제공되며, InnoDB를 통해 트랜잭션 처리와 데이터 무결성을 중시하는 프로젝트에서 더 나은 선택
        금융 IT, 헬스케어 등 안정성이 중요한 분야에서 매우 유용

Maria - 병렬 쿼리 실행과 추가적인 스토리지 엔진으로 대규모 데이터 처리에 유리합니다. 
        실시간 분석과 빠른 응답이 필요한 프로젝트에서 높은 성능을 발휘하며, 데이터 처리 속도를 중시하는 환경에 적합
        최신 기술을 빠르게 도입하고 성능 최적화를 중시하는 프로젝트에 적합
        대규모 데이터를 처리하는 스타트업이나 빠른 개발 주기가 요구되는 프로젝트에서 더 나은 선택
