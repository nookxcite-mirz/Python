
SpringBoot 란?

자바 기반의 애플리케이션, 특히 웹 애플리케이션 및 마이크로서비스를 더 쉽고 빠르게 개발할 수 있도록 도와주는 프레임워크입니다. 
스프링(Spring) 프레임워크의 복잡성을 줄이고, 개발자가 비즈니스 로직에 집중할 수 있게 하는 데 중점을 둡니다.


[핵심 특징]
자동 설정(Auto-configuration): 
    SpringBoot의 가장 큰 장점입니다. 
    프로젝트에 특정 라이브러리(예: 데이터베이스, 웹 서버)가 추가되면, 
    SpringBoot는 개발자가 별도의 설정을 하지 않아도 자동으로 필요한 설정을 구성합니다.

내장된 웹 서버(Embedded Web Server): 
    별도로 Tomcat, Jetty 또는 Undertow 같은 웹 서버를 설치하거나 설정할 필요가 없습니다. 
    SpringBoot 애플리케이션은 스스로 웹 서버를 내장하고 있어, .jar 파일 형태로 실행만 하면 됩니다.

독립 실행 가능(Stand-alone): 
    SpringBoot는 모든 종속성을 포함한 하나의 실행 가능한 .jar 파일을 생성합니다. 
    이 파일만 있으면 어떤 서버든 JVM(자바 가상 머신)이 설치된 환경에서 바로 실행할 수 있습니다.

스타터 의존성(Starter Dependencies): 
    공통적인 기능 묶음을 미리 정의해둔 의존성 집합입니다. 
    예를 들어, spring-boot-starter-web 하나만 추가하면 웹 애플리케이션 개발에 필요한
    모든 라이브러리(Spring MVC, Tomcat 등)가 자동으로 포함됩니다.
